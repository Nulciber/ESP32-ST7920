## Variables globales
```cpp
uint8_t gdram[64][16];
```
C'est le buffer qui est maintenu en RAM. L'écran fait 64 lignes et 16 colonnes de 8 octets chacune (16x8=128 pixels). Il s'agit donc bien d'un tableau de 64 lignes et de 16 colonnes, chaque cellule contenant un octet (8 bits). Ainsi. 64x16x8=8192: les 8192 pixels de l'écran sont adressables.

## function sendCommand
```cpp
void sendCommand(uint8_t cmd)
{
    digitalWrite(CS, HIGH);   // Préparer le ST7920 à recevoir de l'information
    SPI.transfer(0xF8);       // Ce que tu vas recevoir est une commande
    SPI.transfer(cmd & 0xF0); // Voici le premier nibble de l'octet que je t'envoie
    SPI.transfer(cmd << 4);   // Voici le second nibble de l'octet que je t'envoie
    digitalWrite(CS, LOW);    // Le transfert d'information est terminé
    delayMicroseconds(72);    // J'attends que tu aies fini de traiter la commande selon les spécifications de la datasheet
}
```
Le détail des opérations sur bits se trouve dans le fichier notes_ST7920.md

## function sendData
```cpp
void sendData(uint8_t dat)
{
    digitalWrite(CS, HIGH);
    SPI.transfer(0xFA);
    SPI.transfer(dat & 0xF0);
    SPI.transfer(dat << 4);
    digitalWrite(CS, LOW);
    delayMicroseconds(72);
}
```
La fonction sendData se comporte comme la fonction sendCommand. 0xFA indique que ce sont des données qui sont envoyées

## function st7920_init

```cpp
void st7920_init()
{
    //memset est une fonction de string.h. Tous les bits de gdram sont mis à 0;
    memset(gdram, 0, sizeof(gdram)); 
    
    // Initialisation du bus SPI hardware
    // SPI.begin(SCK, MISO, MOSI, SS)
    // MISO = -1 car le ST7920 en mode série est write-only (pas de lecture possible)
    SPI.begin(SCK, -1, MOSI, CS);

    SPI.beginTransaction(SPISettings(
        1000000,     // 1 MHz — prudent, max ST7920 = 2.5 MHz à 4.5V (datasheet p.43)
        MSBFIRST,    // bit de poids fort en premier, comme en bit-bang (i=7 downto 0)
        SPI_MODE3)); // CPOL=1 CPHA=1 : horloge idle HIGH, échantillonnage front descendant
                     // NB: le datasheet indique MODE0 mais le test matériel a confirmé MODE3
    

    pinMode(CS, OUTPUT); // SPI.begin() configure SCK, MOSI et CS automatiquement en OUTPUT
                         // mais par sécurité et robusteste on garde cette ligne car CS est géré manuellement
                         // contrairement à SCK et MOSI qui sont gérés entièrement par le hardware SPI

    digitalWrite(CS, LOW); // On force CS LOW pour s'assurer que le slave ne lit rien pendant l'init

    delay(100); // Stabilisation de l'alimentation
    sendCommand(0x30); delay(2);   // 0x30 = Mode basique.
    sendCommand(0x0C); delay(2);   // 0x0C = Allume l'affichage
    sendCommand(0x01); delay(10);  // 0x01 = Efface la mémoire texte (DDRAM)
    sendCommand(0x30); delay(2);   // 0x30 = Retour mode basique. Recommandation du datasheet
    sendCommand(0x34); delay(2);   // 0x34 = Mode étendu. Passage obligatoire pour accéder au GDRAM
    sendCommand(0x36); delay(2);   // 0x36 = Mode graphique
    // Les delay sont une indication de la datasheet
    // N.B. Pour afficher uniquement du texte, le mode basique (DDRAM) est suffisant.
    // Mais pour travailler au niveau des pixels il faut travailler en mode graphique (GDRAM)
}
```

```cpp
void effacerEcran()
{
    for (int y = 0; y < 64; y++)      // Il faut passer en revue les 64 lignes de l'écran
    { 
        if (y < 32)                   // Moitié haute de l'écran
        {
            sendCommand(0x80 | y);    // Les 32 lignes du haut
            sendCommand(0x80);        // Moitié haute de l'écran
        }
        else
        {
            sendCommand(0x80 | (y - 32)); // Les 32 lignes du bas
            sendCommand(0x88);            // Moitié basse de l'écran
        }
        for (int x = 0; x < 16; x++)      // Pour les 16 colonnes de l'écran (16 x 8 octets)
            sendData(0x00);               // On met les 8 bits à zéro
    }
}
```
